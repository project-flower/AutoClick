AutoClick

概要:
  ウィンドウのボタン等を自動でクリックします。

動作環境:
  Microsoft Windows 10

ファイル:
  AutoClick.exe
    実行ファイル
  AutoClick.ini
    設定ファイル
  AutoClick.config.ini
    動作設定ファイル

使い方:
  1. AutoClick.ini に設定を記述し、AutoClick.exe と同じディレクトリに保存して下さい。

  2. AutoClick.exe を起動して下さい。
     起動すると、タスクトレイにアイコンが表示されます。

  3. AutoClick.exe が起動した状態で設定されたウィンドウが表示されると、
     設定されたコントロールが存在するかどうかが確認されます。
     存在した場合、そのコントロールが自動的にクリックされます。

  4. タスクトレイ アイコンを右クリックすると、コンテキスト メニューが表示され、以下の操作ができます。
     Enabled:
       クリックすると、有効／無効が切り替わります。
       無効にした状態では、チェックが表示されません。
       無効にすると、設定されたウィンドウが表示されても、動作しません。
     Exit:
       AutoClick.exe を終了します。

  5. 有効の状態でタスクトレイ アイコンをクリックすると、その時に設定されたウィンドウが存在すれば、
     自動クリックを試行します。

設定:
  設定ファイル AutoClick.ini の内容は、以下のように記述して下さい。
  日本語を使用する場合は、UTF-16 (Unicode)で保存して下さい。

[ウィンドウのタイトル]
識別子=キャプション

  例えばメモ帳の"置換"ダイアログが表示された時に"キャンセル"をクリックする場合は、以下のようにします。
  識別子は任意の文字列で記述して下さい(※)。

[置換]
Button=キャンセル

  設定は、複数記述することができます。

[置換]
Button=キャンセル

[開く]
Button=キャンセル

  注意:
    ボタン等によっては、アクセラレータ キーが設定されている場合があります。
    例えば"Open"や"開く(O)"のようなボタンは、"O"にアクセラレータ キーが設定されている場合があり、
    その場合は"&Open"や"開く(&O)"のように記述します。

  ※後述 動作設定 クラス名の照合をする場合を除く。

動作設定:
  設定ファイル AutoClick.config.ini の内容を記述し、動作を変更することができます。
  AutoClick.config.ini は、Shift-JIS で保存して下さい。

  内容の例:

[Settings]
CheckClassName=0

  CheckClassName:
    クリックする対象のコントロールのクラス名を照合します。
    右辺の値が 1 の場合、照合を行います。それ以外の場合、照合は行いません。

    照合を行う場合、AutoClick.ini に記述されたコントロールの識別子とクラス名を照合し、
    一致しない場合は、クリックを行わなくなります。
    例えばボタン以外で同じキャプションが設定されているコントロールを対象にしない、といった動作ができます。
    ウィンドウに使われているクラス名は、アプリケーションにより異なります。
    クラス名については、Spy++ 等のツールで調べることができます。
    https://docs.microsoft.com/ja-jp/visualstudio/debugger/spy-increment-help

制限:
  一部の特殊なウィンドウ等、対応できないものがあります。
